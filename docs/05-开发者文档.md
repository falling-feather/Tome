## 1. 项目定位与一句话总览

**Inkless（不存在之书）** 是一个由大语言模型驱动的 **AI 互动叙事 / 文字冒险平台**：
- 后端以 FastAPI 实现一个"叙事引擎 + 多智能体审计 + 五级记忆"的服务；
- 前端用 React + TypeScript 提供"会话列表 + 流式聊天 + 状态面板 + 管理后台"四合一界面；
- 通过 DeepSeek / 硅基流动 双通道 LLM 输出，配合断路器、限流与降级模板，实现可生产部署的小型自托管服务。

## 5. 数据模型（ORM 表结构）

定义于 [backend/app/models.py](../backend/app/models.py)。共 **10 张表**：

| 表名 | 主要字段 | 说明 |
|------|---------|------|
| `users` | id, username (uniq), password_hash, is_admin, created_at | bcrypt 哈希 |
| `game_sessions` | id (UUID), user_id, title, scenario, **state(JSON)**, created_at, updated_at | `state` 是整个游戏运行时状态 |
| `messages` | id, session_id, role(user/assistant/system), content, **metadata(JSON)**, created_at | metadata 存 segments/event/items/audit |
| `activity_logs` | id, user_id, username, action, detail, ip_address, user_agent, created_at | 由路由 + 中间件写入 |
| `user_api_keys` | id, user_id, provider(deepseek/siliconflow), api_key, base_url, model | 覆盖全局配置 |
| `game_events` | id, event_key(uniq), category, title, description, conditions(JSON), base_weight, cooldown_turns, effects(JSON) | 预留表，目前事件以代码常量定义于 `EVENTS` |
| `world_entries` | id, scenario, layer(core/chapter/ephemeral), category, title, keywords, content, chapter_min/max, priority, is_active | 世界书 |
| `prompt_templates` | id, name, scenario, version, content, is_active | 版本化模板 |
| `memory_entries` | id, session_id, level(1–5), content, turn_start/end, chapter, token_estimate, metadata(JSON) | 五级记忆 |
| `custom_stories` | id, user_id, title, raw_content, parsed_data(JSON), status(pending/parsing/ready/error), error_msg | 自编故事 |

> ⚠️ ORM 字段名 `metadata_` 在数据库里映射为 `metadata` 列（避开 SQLAlchemy 保留属性）。

### `GameSession.state` 的 schema（运行时关键）


## 6. HTTP / SSE 接口规范

所有接口前缀 `/api`，鉴权方式：`Authorization: Bearer <jwt>`。

### 6.1 Auth — `/api/auth`

| 方法 | 路径 | 请求体 | 响应 | 备注 |
|------|------|--------|------|------|
| POST | `/register` | `{username, password}` | `TokenResponse` | username 2–64，password 6–128 |
| POST | `/login` | `{username, password}` | `TokenResponse` | 失败也写日志 |
| GET | `/me` | — | `UserInfo` | 当前用户 |

### 6.2 Game — `/api/game`

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/sessions` | 创建会话；可传 `story_id` 进入自编故事模式 |
| GET | `/sessions` | 当前用户的会话列表（按 `updated_at` desc） |
| GET | `/sessions/{id}` | 返回 `{messages, state}` |
| PATCH | `/sessions/{id}` | 改名 `{title}` |
| DELETE | `/sessions/{id}` | 删除（级联消息） |
| **POST** | **`/sessions/{id}/action`** | **核心：SSE 流式提交玩家动作** |
| GET | `/sessions/{id}/export` | 下载 Markdown 冒险日志 |

**SSE 数据格式**（`text/event-stream`，每行 `data: <json>\n\n`）：

```jsonc
// 流中：每收到一段 LLM token
{"content": "你推开沉重的木门……"}

// 末尾：包含最终状态与段落
{"done": true, "state": { ... }, "segments": [
   {"speaker":"narrator","content":"…"},
   {"speaker":"酒馆老板","content":"…"}
]}

// 错误：
{"content":"【系统】叙事生成遇到问题…","done":true,"error":true}
```

### 6.3 Stories — `/api/stories`

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `` | 上传故事原文（≥50 字、≤50000 字），后台异步解析 → ready |
| GET | `` | 列表 |
| GET | `/{id}` | 详情（含 parsed_data） |
| DELETE | `/{id}` | 删除 |

解析完成后会自动写入 `world_entries`（scenario = `custom_<id>`），并在创建会话时被 `GameEngine` 当作场景。

### 6.4 Settings — `/api/settings`

| 方法 | 路径 | 说明 |

| GET | `/apikeys` | 列出两通道当前生效配置（密钥已脱敏 `sk-***xxxx`） |
| PUT | `/apikeys` | 更新某 provider 的 key/base_url/model |
| DELETE | `/apikeys/{provider}` | 移除用户级覆盖 |

### 6.5 Admin — `/api/admin`（`require_admin` 守卫）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/stats` | 用户/会话/消息/日志总数 + 最近 10 注册 |

## 7. 核心服务模块详解

### 7.1 GameEngine — `services/game_engine.py`

- **6 状态**：exploring / combat / resting / trading / dialogue / dead；
- **VALID_TRANSITIONS** 定义了允许的状态跃迁；
- **事件池 `EVENTS`**：12 通用 + 3 奇幻 + 3 科幻 + 3 武侠 = **21 条事件**，每条带 `conditions`(min_turns / min_chapter / min_health / min_money / max_health_pct / require_status / require_flag)、`weight`、`cooldown`、`effects`、`scenarios`；
- **动态无事件概率**：前 10 回合 30%，之后 15%；
- **结算器 `update_state`** 负责：回合 +1、冷却衰减、应用事件 effects、推断状态、章节自动推进等。

关键 API：

| 方法 | 用途 |
|------|------|
| `create_initial_state(name, cls, scenario)` | 生成 state JSON |
| `generate_intro(state)` | 默认场景开场白 |
| `validate_action(state, text)` → `(bool, reason)` | 死亡 / 高疲劳战斗等约束 |
| `check_events(state)` → `(narrative, log)` | 加权抽取一个事件，并返回完整候选/跳过日志 |
| `update_state(state, user_text, ai_text, event_log)` | 一次回合的最终状态写入 |

### 7.2 LLMService — `services/llm_service.py`

- 优先 DeepSeek，失败则降级 SiliconFlow；
- `load_user_keys()` 读取 `user_api_keys` 覆盖全局；
- `stream_narrative()`：组装 system prompt（PromptAssembler + WorldBook + Memory 检索）→ httpx 流式 → 触发 `llm_circuit_breaker` / `llm_fallback_circuit_breaker`；
- `simple_completion()` 给 Agent / Memory / 故事解析等"工具型"调用使用；
- 所有耗时通过 `health_metrics.record_timing("llm_stream_ms", ...)` 记录。

### 7.3 PromptAssembler — `services/prompt_assembler.py`

六层装配（动静分离，由 DB 模板组合）：

```
system_core  →  style_<scenario>  →  world_lore (TopK)  →  current_state
            →  status_<status>    →  event_context
```

模板存于 `prompt_templates` 表，**更新走"新建更高 version + 老版本 is_active=False"**，便于回滚。

### 7.4 WorldBookService — `services/world_book.py`

- 分层 `core / chapter / ephemeral`；
- 检索：根据上下文文本做关键词命中评分 + chapter 范围过滤 + priority 排序，返回 TopK；
- 内置约 25 条种子（通用规则 + 三个场景核心设定 + 章节剧情铺垫）。

### 7.5 MemoryService — `services/memory_service.py`

| 级别 | 触发 | 内容 | Token 预算 |
|------|------|------|-----------|
| L1 | 直接读 `messages`（最近 8 条） | 原始消息 | — |
| L2 | 消息数 > 16 | 10 条压一段摘要 | 300 |
| L3 | `chapter` 自增 | L2 → 章节摘要 | 200 |
| L4 | 累计 ≥ 3 个 L3 | 弧线摘要 | 150 |
| L5 | 累计 ≥ 2 个 L4 | 永久核心记忆 | 100 |

`retrieve_context(max_chars=2000)` 倒序拼接 L5→L4→L3→L2 注入到 system prompt 末尾。

### 7.6 ResponsePostProcessor — `services/post_processor.py`

- 移除元叙事（"作为 AI"等）、移除 "你可以…" 选项列表、剥离 Markdown；
- 用正则提取 物品获取/失去、NPC 名、金币变动；
- 解析 `[旁白]` / `[角色:酒馆老板]` 段落标签，前端据此分气泡渲染。

### 7.7 Agents — `services/agents.py`

四个独立审查 + 一个 Orchestrator：

| Agent | 关注点 | 触发动作 |
|-------|--------|---------|
| CharacterAgent | 职业能力 / 场景元素错配 / 角色名误用 | 记 issue |
| WorldAgent | 死亡状态行为、地点关键词、HP/疲劳描写一致性 | 记 issue |
| NarrativeAgent | AI 身份泄露、长度、视角、套路、Markdown 残留 | 记 issue |
| AuditAgent | 当前三者 issue 总数 ≥ 阈值时调用 LLM 重写 | 写 `rewrite` |
| **AgentOrchestrator** | 汇总 verdicts，决定是否替换最终文本 | 返回 `OrchestratorResult` |

`audit_result.was_rewritten` 会进入消息 metadata，前端可见。

### 7.8 Resilience — `services/resilience.py`

- **CircuitBreaker**：CLOSED→OPEN→HALF_OPEN，失败阈值 5 / 30s 恢复窗口；
- **RetryPolicy**：指数退避（最多 2 次，base 1s，max 8s）；
- **RateLimiter**：滑动窗口，**单用户 15/min，全局 200/min**；
- **HealthMetrics**：counters / gauges / timings(p50/p95/p99/max)；
- 全部以 **模块级单例** 暴露：`llm_circuit_breaker`、`llm_fallback_circuit_breaker`、`game_rate_limiter`、`health_metrics`。

### 7.9 StoryImportService — `services/story_import.py`

LLM 解析用户故事 → JSON `{title_suggest, plot_summary, world_rules, opening_scene, characters[≤8], locations[≤6]}` → 写入 `world_entries(scenario=custom_<id>)`，每用户上限 20 个故事，原文最长 50000 字。

## 8. 游戏运行时数据流

一次玩家行动的全链路（`POST /api/game/sessions/{id}/action`）：

```
client ──SSE──▶ FastAPI route(game.submit_action)
   1. RateLimiter.check(user_id) ─┐失败→429
   2. health_metrics.inc("game_actions")
   3. GameSession 取 state, 校验 validate_action
   4. 写入 user Message
   5. GameEngine.check_events(state) → (event_narrative, event_log)
   6. LLMService.stream_narrative(state, history, action, event_narrative, session_id)
        ├─ build_system_prompt:
        │     WorldBookService.retrieve(scenario, chapter, ctx, top_k=5)
        │     MemoryService.retrieve_context(max_chars=1500)
        │     PromptAssembler.assemble(state, lore, event_context)
        ├─ CircuitBreaker.can_execute → DeepSeek httpx 流
        └─ 失败时 fallback CircuitBreaker → SiliconFlow
   7. 边收边 yield "data: {content}\n\n"
   8. 收完后:
        ResponsePostProcessor.process(text, state) → cleaned + extracted + segments
        AgentOrchestrator.process(cleaned, state, llm.simple_completion)
            └─ 若 was_rewritten 则替换 cleaned，并重新提取
        GameEngine.update_state(state, user_text, cleaned, event_log)
        合并 inventory / money（裁剪到上限）
        MemoryService.check_and_compress(state, llm.simple_completion)
        写 assistant Message（带 metadata: segments/event/items/npcs/audit/warnings）
   9. yield "data: {done:true,state,segments}\n\n"
   10. ActivityLog 写入
```

## 9. 前端模块结构
