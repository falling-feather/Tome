# 详细工作计划与里程碑

## 1. 计划目标
在 12 周内完成从可玩原型到可上线版本的核心能力建设，确保系统具备：

1. 可玩性：稳定的互动叙事闭环
2. 一致性：角色与世界观不漂移
3. 稳定性：可恢复、可降级、可观测
4. 可扩展性：后续可接入更多模型与内容

## 2. 团队建议配置

- 后端工程师 2 名
- 前端工程师 1 名
- AI 应用工程师 1 名
- 测试工程师 1 名（可兼职）
- 产品/叙事策划 1 名

## 3. 12 周分阶段计划

## 阶段 A（第 1-2 周）：基础骨架与最小闭环

### 目标
打通最小会话主循环。

### 任务
1. 建立 FastAPI 工程骨架
2. 初始化 PostgreSQL、Redis 与迁移工具
3. 设计会话与状态核心表
4. 实现接口：创建会话、提交行动、查询状态
5. 接入 LLM 网关（单模型）
6. 完成基础日志与错误码规范

### 验收标准
1. 单轮交互全链路成功率 >= 95%
2. 每轮请求有完整 trace 与日志
3. 可完成 20 轮连续对话不崩溃

## 阶段 B（第 3-4 周）：状态机与事件池

### 目标
建立可控叙事骨架。

### 任务
1. 实现显式状态机与变量结算器
2. 实现事件池、前置条件、冷却与权重
3. 事件触发日志落库（含候选与未触发原因）
4. 完成基础动作校验与非法迁移拦截

### 验收标准
1. 状态迁移可追踪、可回滚
2. 100 轮仿真无死循环
3. 事件重复率低于预设阈值（建议 <= 25%）

## 阶段 C（第 5-6 周）：RAG 世界书与 Prompt 装配

### 目标
建立设定增强能力并控制上下文质量。

### 任务
1. 世界书分层建模（核心/章节/短期）
2. 向量索引与混合检索实现
3. Prompt 模板管理与版本化
4. 动静分离装配器落地
5. 输出格式校验与后处理管道

### 验收标准
1. 检索命中相关性达标（人工+自动评测）
2. 角色设定冲突率明显下降
3. Prompt 快照可用于问题复盘

## 阶段 D（第 7-8 周）：五级记忆压缩与恢复

### 目标
支持长程会话并避免 Token 超限。

### 任务
1. 实现 L1-L5 压缩触发策略
2. 实现压缩后关键节点回灌
3. 实现压缩一致性校验与回退
4. 处理 413 与超限异常路径

### 验收标准
1. 200 轮长会话可稳定运行
2. 关键事实保留率 >= 95%
3. 超限恢复成功率 >= 95%

## 阶段 E（第 9-10 周）：多智能体编排与审计

### 目标
提升叙事一致性与可控性。

### 任务
1. 实现角色代理、世界观代理、叙事代理、审计代理
2. 实现主编排器仲裁与候选合成
3. 实现规则冲突拦截与重写策略
4. 加入降级输出模板

### 验收标准
1. 角色漂移率下降到目标区间
2. 规则违规输出显著减少
3. 超时场景下仍可返回可用结果

## 阶段 F（第 11-12 周）：稳定性加固与上线准备

### 目标
达到可上线运行标准。

### 任务
1. 断路器、重试、限流、超时策略联调
2. 建立 Prometheus 指标与 Grafana 看板
3. 完成压测、故障演练、回归测试
4. 输出上线检查清单与应急预案

### 验收标准
1. P95 响应 <= 8 秒
2. 接口成功率 >= 99%
3. 断路恢复成功率 >= 95%
4. 关键告警可在 5 分钟内定位

## 4. Sprint 执行节奏（建议）

- Sprint 周期：2 周
- 每日站会：15 分钟
- 评审机制：PR 双人 Review
- 质量门禁：单测 + 集成 + 基础 E2E
- 发布节奏：每 Sprint 至少一次可演示版本

## 5. 任务拆解模板（单 Sprint）

1. 需求拆分：用户故事 + 验收标准
2. 技术设计：接口、数据结构、异常路径
3. 开发实施：先主链路，再异常链路
4. 测试验证：功能、回归、性能
5. 文档沉淀：接口文档、运行手册、变更记录

## 7. 风险清单与应急预案

1. LLM 成本超支
- 预案：模型分级降级、缓存优先、长度限制

2. 角色漂移
- 预案：静态规则置顶、审计代理拦截、回灌关键节点

3. 上下文超限
- 预案：五级压缩 + Reactive 紧急回收

4. 模型调用抖动
- 预案：重试、断路、降级模板兜底

5. 存档异常
- 预案：快照版本管理 + 校验 + 回滚

## 8. 质量与观测清单

上线前必须满足：

1. 单元测试覆盖核心模块

3. 长会话压力测试通过
4. 指标、日志、追踪全部可用
5. 告警策略可触发且可恢复
