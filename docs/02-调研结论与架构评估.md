# 调研结论与架构评估

## 1. 调研目标
结合当前项目定位，对系统设计进行工程化可行性评估，输出可执行的模块边界、技术选型与上线风险控制策略。

## 2. 核心结论（可直接采用）

1. 你的基础框架方向正确，关键在于把规则控制放到后端显式系统
2. 多智能体价值在审计与纠偏，不在于无约束并行对话
3. 五级上下文机制是核心壁垒，必须配合回灌与回退机制落地
4. RAG 必须分层并重排，否则会出现设定噪声与 Token 浪费
5. 首选 FastAPI 路线，迭代效率与生态更适合此类 AI 应用

## 3. 推荐模块拆分与职责

### 3.1 核心模块

- StateEngine：显式状态机与变量结算
- EventPool：事件触发、冷却、权重计算
- PromptManager：模板管理、上下文装配、版本追踪
- MemoryCompressor：五级压缩与恢复策略
- KnowledgeRetriever：世界书检索、重排、注入
- AgentOrchestrator：多代理协作与仲裁
- LLMGateway：模型调用、缓存、超时、重试
- OutputGuard：格式校验、越界过滤、术语清洗
- SessionStore：会话与快照持久化
- Observability：日志、指标、Trace、告警

### 3.2 模块间关键依赖

- StateEngine -> EventPool：按状态决定候选事件
- EventPool -> PromptManager：将客观事件注入上下文
- KnowledgeRetriever -> PromptManager：世界书条目注入
- MemoryCompressor -> PromptManager：预算内历史压缩
- PromptManager -> LLMGateway：统一请求模型
- LLMGateway -> OutputGuard：结果后处理
- OutputGuard -> SessionStore：安全输出落库

## 4. 技术选型评估

### 4.1 后端与存储

- FastAPI：异步性能与开发效率平衡最好
- PostgreSQL：强一致事务 + JSONB + 审计友好
- Redis：会话缓存、限流、热模板缓存
- Vector DB：Milvus（规模化）或 pgvector（低成本起步）

### 4.2 AI 层

- 模型网关封装：支持多供应商切换
- Embedding 与生成解耦：便于成本与效果分别优化
- 强制结构化输出：降低后处理复杂度

### 4.3 运维与观测

- Prometheus + Grafana：指标可视化
- OpenTelemetry：全链路追踪
- 结构化日志：可按 session_id 与 trace_id 回放

## 5. 架构策略评估

### 5.1 动静分离 Prompt（必要）

- 静态层必须置顶且固定
- 动态层按预算注入
- 请求级快照可支持线上复盘

收益：

- 角色稳定性提升
- 抗越权能力增强
- 调试效率显著提高

### 5.2 五级压缩（必要）

触发建议：

- 预算 60%：L1
- 预算 70%：L2
- 预算 80%：L3
- 预算 90%：L4
- 413/超限异常：L5

关键补充：

- 压缩后一致性校验
- 校验失败回退上一压缩版本
- 强制回灌关键节点与人设规则

### 5.3 事件池（建议尽早实现）

- 用事件权重日志支持运营调参
- 冷却机制防止短期重复
- 可解释触发原因支持剧情调优

## 6. 测试与评估框架

### 6.1 测试分层

- 单元测试：状态转移、权重计算、压缩触发
- 集成测试：会话管线、RAG 注入、模型调用
- 端到端测试：长对话稳定性、恢复机制、读档一致性

### 6.2 建议重点指标

- 延迟：P95 <= 8 秒
- 稳定性：成功率 >= 99%
- 一致性：角色/世界观一致性 >= 90%
- 成本：单轮 Token 成本可观测、可降级
- 压缩有效性：关键事实保留率 >= 95%

## 7. 风险评估与治理建议

### 7.1 高风险项

1. Token 成本失控
2. 角色漂移与设定冲突
3. 上下文溢出导致响应失败
4. 事件重复导致可玩性下降
5. 模型超时导致体验抖动

### 7.2 对应治理

- 预算治理：模型分级、响应长度上限、缓存命中优化
- 质量治理：输出审计代理 + 结构化规则校验
- 稳定治理：断路器、重试、降级模板
- 内容治理：事件冷却、频次限制、多样性约束

## 8. 实施建议（结论）

1. 先打通主循环，再扩展多智能体
2. 先上最小 RAG，再扩展世界书规模
3. 上下文压缩优先做机制闭环，再做策略精调
4. 指标体系与日志体系必须与功能并行建设

## 9. 当前建议落地顺序

1. 最小主循环 + 状态机 + 持久化
2. 事件池 + 基础 RAG
3. 五级压缩 + 回灌恢复
4. 多智能体审计编排
5. 观测、压测与生产加固

以上结论可直接作为架构评审输入与技术实施依据。
